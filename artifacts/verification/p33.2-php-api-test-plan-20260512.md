## Adım 2: ASSET_TOKEN Tanımlama

İki yöntem var. Sırasıyla dene:

### Yöntem 1 (Öncelikli): .htaccess SetEnv

`albago.tr/.htaccess` dosyasında şu satırı aktif et:

```apache
SetEnv ASSET_TOKEN "your-secret-token-here"
```

`your-secret-token-here` yerine kendi belirlediğin token'ı yaz. En az 32 karakter, rastgele.

`config.php` bu değeri `getenv('ASSET_TOKEN')` ile okur.

**Test:** `.htaccess`'i güncelle, health endpoint'i çağır. 200 dönüyorsa SetEnv çalışıyor.

### Yöntem 2 (Yedek): Private config dosyası

Eğer SetEnv çalışmazsa (health 500 + `ASSET_TOKEN not configured` dönerse):

1. `albago.tr/data/` içinde `asset-secret.php` dosyası oluştur:

```php
<?php
// Bu dosya data/.htaccess ile public erisime kapatilmistir.
return 'your-secret-token-here';
```

2. `your-secret-token-here` yerine kendi token'ını yaz.

3. `data/.htaccess` zaten `Require all denied` içeriyor — web erişimi kapalı.

`config.php` önce getenv dener, boşsa `data/asset-secret.php` dosyasını include eder. İkisi de yoksa 500 ile fail eder.

**Test:** Dosyayı oluştur, health endpoint'i çağır. 200 dönüyorsa çalışıyor.

### Token Kuralları

- Gerçek token koda yazılmaz (`.htaccess` veya `asset-secret.php` içinde)
- Default/fallback token yok
- Token yoksa API 500 fail
- Token Claude Code'a ve frontend'e girmez
- Aynı token P33.3'te `backend/.env` → `PHP_ASSET_TOKEN` olarak kullanılacak

### Doğrulama

Token tanımlandıktan sonra health endpoint çalıştırılır (Adım 3).

## Adım 5: Geçerli Upload → 201

### Test 5.1: PNG upload

Gerçek bir PNG dosyası ile test et (en az 100 byte gerçek PNG).

```bash
curl -s -w "\nHTTP_STATUS:%{http_code}\n" \
  -X POST \
  -F "file=@gercek_resim.png" \
  -F "category=covers" \
  -H "Authorization: Bearer <ASSET_TOKEN>" \
  https://albago.tr/api/assets/upload | python -m json.tool
```

**Beklenen:** HTTP 201, JSON:
```json
{
    "id": "a1b2c3d4e5f6g7h8",
    "filename": "gercek_resim.png",
    "url": "https://albago.tr/assets/covers/gercek_resim.png",
    "category": "covers",
    "mimeType": "image/png",
    "bytes": ...,
    "sha256": "...",
    "width": ...,
    "height": ...,
    "version": 1,
    "archived": false,
    "createdAt": "2026-05-12T..."
}
```

**PASS kriteri:** 201, JSON'da `id` 16 karakter hex, `url` `https://albago.tr/assets/covers/` ile başlıyor, `archived: false`

## Adım 8: Archive Endpoint

### Test 8.1: Archive (soft-delete)

Yüklenen bir asset'in `id` değerini al (list endpoint'ten veya upload yanıtından):

```bash
ASSET_ID="<upload-yanitindaki-id>"

curl -s -w "\nHTTP_STATUS:%{http_code}\n" \
  -X POST \
  -H "Authorization: Bearer <ASSET_TOKEN>" \
  https://albago.tr/api/assets/$ASSET_ID/archive | python -m json.tool
```

**Beklenen:** HTTP 200, `{"archived": true, "id": "..."}`

**PASS kriteri:** 200, `archived: true`

### Test 8.2: Archive edilen asset listede görünmez

```bash
curl -s -H "Authorization: Bearer <ASSET_TOKEN>" \
  "https://albago.tr/api/assets" | python -m json.tool
```

**PASS kriteri:** Archive edilen asset `items` listesinde yok (varsayılan `archived=false` filtresi).

### Test 8.3: Archive edilen asset archived=true ile görünür

```bash
curl -s -H "Authorization: Bearer <ASSET_TOKEN>" \
  "https://albago.tr/api/assets?archived=1" | python -m json.tool
```

**PASS kriteri:** Archive edilen asset listede görünüyor, `archived: true`.

### Test 8.4: Asset URL hala erişilebilir

Archive edilen asset'in `url` değerini browser'da aç.

**PASS kriteri:** Dosya fiziksel olarak silinmemiş, URL hala çalışıyor.

### Test 8.5: Aynı asset tekrar archive → idempotent

```bash
curl -s -X POST \
  -H "Authorization: Bearer <ASSET_TOKEN>" \
  https://albago.tr/api/assets/$ASSET_ID/archive
```

**PASS kriteri:** 200, hata yok (zaten archived olan asset tekrar archive edilebilir).

### Test 8.6: Geçersiz ID ile archive

```bash
curl -s -w "\nHTTP_STATUS:%{http_code}\n" \
  -X POST \
  -H "Authorization: Bearer <ASSET_TOKEN>" \
  https://albago.tr/api/assets/invalid-nonexistent-id/archive
```

**PASS kriteri:** 404, `asset_not_found`
