### Backend (3 findings)

| # | Package | Severity | Direct/Transitive | Affected workspace | Fix available | Breaking change risk |
|---|---------|----------|-------------------|-------------------|---------------|---------------------|
| 1 | effect | high | transitive | backend | yes (`npm audit fix`) | None — semver-compatible fix exists |
| 2 | @prisma/config | high | transitive | backend | yes (via effect fix) | None |
| 3 | prisma | high | transitive | backend | yes (via effect fix) | None |

**Chain:** `effect <3.20.0` → `@prisma/config` → `prisma`

**Vulnerability:** Effect `AsyncLocalStorage` context lost/contaminated inside Effect fibers under concurrent load with RPC (GHSA-38f7-945m-qr2g).

**Runtime exposure:** The backend uses Prisma for database access. The vulnerable `effect` package is a dependency of `@prisma/config`. The advisory describes fiber context contamination under concurrent load with RPC. AlbaGo backend:
- Does NOT use Effect fibers directly
- Does NOT use Effect RPC
- Uses Prisma via NestJS services (not concurrent fiber patterns)
- The vulnerability path requires specific Effect concurrent patterns that are not in use

**Decision:** NOT a release blocker. Fix is available and non-breaking; can be applied after RC packaging.

## Decision

- **Release blocker:** NO
- **Reason:** 
  - Backend effect vulnerability is transitive through Prisma and has no practical exploit path in the current architecture (no Effect fiber/RPC usage).
  - Admin Next.js vulnerabilities primarily affect public-facing production deployments; the admin panel is an internal tool.
  - The available fix for Next.js requires a breaking major version downgrade (`--force`), which is higher risk than the vulnerabilities themselves.
  - Backend fix is non-breaking and can be applied post-RC.
- **Follow-up:** 
  1. Apply `npm audit fix` in backend workspace after RC is packaged (non-breaking semver fix for effect).
  2. Monitor Next.js releases for a patched 15.x version and upgrade when available.
  3. If admin panel becomes public-facing, re-triage Next.js CVEs as potentially blocking.
