## Changes

### P0 — Verification Environment
- Created `scripts/verify-platform-v2.ps1` — deterministic local verification script

- Updated `gradle.properties` with `org.gradle.workers.max=1`

### P1 — Android Program Runner v1
- **Files modified**: `android/core_runtime/src/main/java/com/alba/core/runtime/GameRuntime.kt`
- **Files created**: `android/core_runtime/src/test/java/com/alba/core/runtime/GameRuntimeProgramRunnerTest.kt`
- Added `ProgramStepStatus` enum (NOT_STARTED, ACTIVE, COMPLETED, BLOCKED)
- Added `ProgramRuntimeStepState` and `ProgramRuntimeState` data classes
- Extended `GameRuntimeState` with `program: ProgramRuntimeState` field
- Implemented step progression state machine in `GameRuntime`:
  - `start(nowMs)` — initializes program from `level.programSteps`, marks first step active
  - `tick(nowMs)` — updates timer-based steps (REST, INSTRUCTION, HOLD_POSE, PLAY_GAME)
  - `onMotionEvent(event)` — program runner handles MOTION_REPS; templates continue for PLAY_GAME
  - Step completion auto-advances (unless `nextOnComplete = false`)
- **Step behaviors**:
  - `PLAY_GAME`: Template-based gameplay, optional `durationSec` for auto-completion
  - `MOTION_REPS`: Advances on matching `REP_COUNTED`, completes at `targetCount`
  - `REST`: Timer-gated, no scoring, completes after `durationSec`
  - `INSTRUCTION`: Auto-advances after `durationSec` (v1 fallback: 3s if absent)
  - `HOLD_POSE`: Timer-gated hold v1, pauses on `USER_OUT_OF_FRAME`, resumes continue timer
- Added 10 unit tests covering all step types and edge cases

### P2 — GameDefinition v3 Contract Alignment
- **Backend**: Updated `backend/src/common/game-definition-v3.ts`
  - `REST` now requires `durationSec > 0` (was optional)
  - `MOTION_REPS` now requires `motion` field
  - `INSTRUCTION` without `durationSec` generates warning (v1 defaults to 3s)
  - Added step `title` validation
- **Backend tests**: Added 8 new test cases for program step validation (16 total)
- **Android parser**: `V3ProgramStepDefinition` now includes `successMessage` field
  - `parseStep()` now parses and stores `successMessage`

### P3 — Camera Readiness & QA UX
- **Files modified**: `android/feature_games/src/main/java/com/alba/feature/games/GamesFeature.kt`
- Added `CameraReadinessCard` composable with guidance for:

  - `HAND_TARGET`: Hands/wrists visible, sufficient light
