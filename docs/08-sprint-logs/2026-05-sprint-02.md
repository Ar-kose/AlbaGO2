# Sprint 02 Log

## Goal

Move AlbaGo from architecture scaffold to a real Android vertical slice with camera, pose, motion counting, `TARGET_HIT` and early device-readiness tooling.

## What was built

- Visible product name updated from `Alba` to `AlbaGo`
- CameraX preview, MediaPipe pose estimation and pose overlay kept live and buildable
- Motion controller extended with QA backend override, cached game fallback and sync retry queue
- Workout and game sessions now use stable client session keys
- Motion debug UI now exposes detector, sync and backend diagnostics for QA
- Squat and jumping jack detectors were tuned with stricter state machines and timing gates
- Jump rope detector moved to a baseline-calibrated first prototype
- Backend gained hybrid Prisma persistence scaffolding for game definitions and audit logs
- Admin panel remained focused on AlbaGo `TARGET_HIT` content editing and publish flow

## Files and modules touched

- Android `app`, `core_data`, `core_motion`, `core_network`, `feature_workout`, `feature_games`
- Backend `main`, `workouts`, `game-sessions`, `games`, `content-publish`, `audit-logs`, `persistence`, `prisma`
- Admin `layout`, `page`, `games` console and API client
- README, OpenAPI and release documentation set

## API and interface changes

- Android create-session requests now send `clientSessionKey`
- Game update payloads now carry `gameVersion`
- Workout motion summary now includes motion type and detector version
- Backend session create endpoints are idempotent by `clientSessionKey`
- Prisma schema and migration files now exist for PostgreSQL-backed persistence

## Test status

- `npm.cmd run prisma:generate --workspace backend`: passed
- `npm.cmd run build --workspace backend`: passed
- `npm.cmd run test --workspace backend`: passed
- `npm.cmd run build --workspace admin`: passed
- `android/gradlew.bat clean assembleDebug`: passed
- `android/gradlew.bat testDebugUnitTest`: blocked by local Gradle test worker environment issue

## Android device verification

- Real APK build succeeded again after Sprint 3 changes.
- A physical Android device was detected with `adb`, the debug APK was installed successfully and the app cold-launched through `adb shell am start`.
- `adb reverse tcp:3000 tcp:3000` was also verified for USB-backed backend testing.
- Manual camera, pose, motion and gameplay walkthrough on the device is still pending, so full Sprint 3 acceptance is not closed yet.

## Known issues

- `JUMP_ROPE` remains prototype quality.
- Android JVM unit tests are blocked by the local Gradle worker environment.
- Prisma persistence requires a configured `DATABASE_URL`; otherwise fallback remains in-memory.

## Admin panel mapping

- AlbaGo branding is visible in admin metadata and dashboard copy.
- Draft save, validation, publish, rollback and audit visibility remain available for `TARGET_HIT`.

## Remote config readiness

- AlbaGo Android now prefers `network -> cached published definition -> debug fallback`.
- QA can change backend URL without rebuilding the app.

## Rollback path

- Admin can still move a published game back to `REVIEW`.
- If backend content is unavailable, debug builds can continue with cached or built-in fallback content.

- Capture a short AlbaGo demo video.

- Run Prisma migrations against local PostgreSQL and validate persisted publish/audit flows.
