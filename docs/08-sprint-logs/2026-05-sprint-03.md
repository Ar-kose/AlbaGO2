# Sprint Log — 2026-05 Sprint 03

## Summary

Sprint 4 focused on turning AlbaGo into a presentation-ready Android demo with 3 public games and stronger real-device evidence.

## What shipped

- Android home surface was redesigned into an `arcade sport` presentation shell.
- Home copy was tightened toward product-demo language and user-facing debug/backend status was removed from game catalog cards.
- Motion Lab now hides debug HUD by default and exposes QA controls on demand.
- Public game catalog now lists:
  - `FRUIT_SLASH`
  - `DODGE_RUN`
  - `FIT_CHALLENGE`
- Game runtime now supports template-specific scene states, combo, accuracy and structured result payloads.
- Catalog crash hardening was added:
  - removed nested game-screen scroll
  - replaced unsafe `levels.first()` use in catalog/detail/HUD paths
  - safely parses unknown remote/cache enum values
  - filters invalid definitions before rendering or start
- Backend and Prisma were extended for multi-template game definitions with `config` and `tasks`.
- Admin console now edits multiple public templates instead of a single `TARGET_HIT` flow.
- Admin-driven game definition v2 now includes `orientation`, `cameraRequirement`, typed `assets.items[]`, `sceneConfig` and `interactionRules`.
- Backend now supports local-dev asset upload and serving for PNG, WebP and sanitized SVG assets.
- Admin can upload assets, choose portrait/landscape behavior and inspect the exact mobile payload before publish.
- Android applies game orientation during active play and resolves typed uploaded assets with placeholder fallback.
