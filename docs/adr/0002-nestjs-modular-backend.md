# ADR-0002: NestJS Modular Monolith Backend

## Durum

Kabul edildi.

## Bağlam

MVP hızına uygun, ama ileride modül ayrıştırmasına açık bir backend gerekiyor.

## Karar

Backend, NestJS modular monolith olarak başlatılacak.

## Sonuçlar

İlk sprintte geliştirme hızı artar. Servisleşme ihtiyacı doğduğunda modül sınırları korunmalıdır.
